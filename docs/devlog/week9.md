# Week 9 — 폴리싱/LLM/버퍼 (추천 4~6시간)

목표: HUD/토스트 폴리싱, 심문 문구 템플릿, 선택적으로 LLM HTTP 스텁을 적용하고, 밀린 작업을 회수한다.

## 해야 할 일
- UI 폴리싱
  - HUD 정렬/패딩, 폰트 크기, 색상 일관화.
  - 토스트/로그에 아이콘 혹은 배경 투명도 적용해 가독성 개선.
  - 해상도별 Canvas Scaler 동작 확인.
- 문구/LLM
  - 심문 문구 템플릿 파일을 정리하고 플레이스홀더를 실제 데이터로 치환.
  - 선택: 간단한 HTTP 스텁 작성(`POST /llm` 등) 후 실패 시 템플릿 폴백을 확인.
- 버퍼 작업
  - 이전 주 미완료 항목 2개 이상 해소(예: NavMesh 튜닝, Report 쿨다운, 토스트 큐 중복 방지 등).
  - 자잘한 경고/콘솔 로그 제거.
- 체크리스트
  - Canvas Scaler `Match Width or Height` 값 조정 후 16:9 / 16:10 / 4:3에서 UI 깨짐 여부 확인.
  - 색상/폰트/패딩을 프리셋(ScriptableObject나 UI Style Asset)으로 묶어 중복 설정을 줄인다.
  - LLM 스텁은 타임아웃/에러 시 템플릿으로 폴백하는지 로그로 검증.

## DoD
- UI가 목표안에 근접하고 심문 문구 템플릿/LLM 폴백이 동작.
- 밀린 TODO를 최소 2개 이상 정리(기록 남기기).
